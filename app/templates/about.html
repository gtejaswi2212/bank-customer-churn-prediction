About This Project

Dataset, approach, and technical choices.

Dataset

We use the analytical_base_table.csv (10,000 rows) with customer demographics and product usage: credit score, geography, gender, age, tenure, balance, number of products, credit card ownership, active membership, estimated salary, and the target Exited (churn).

Approach

The pipeline is built for reproducibility and clarity: load → validate → engineer features → encode categoricals → scale → split (train/val/test) → handle class imbalance (SMOTE) → train multiple models → select best by ROC-AUC → evaluate on test set → save artifacts and plots.

Preprocessing

  • Missing values filled with median (numeric) or mode (categorical).
  • Categorical variables (Geography, Gender) one-hot encoded with drop-first.
  • Numeric features standardized with StandardScaler (fit on train only).
  • Optional derived features: balance-to-salary ratio, tenure-to-age ratio, multi-product flag.

Modeling

We compare Logistic Regression, Random Forest, and XGBoost on a validation set. The model with the highest ROC-AUC is chosen and evaluated on the held-out test set. Class imbalance is addressed with SMOTE on the training set.

Evaluation

Metrics reported: Accuracy, Precision, Recall, F1, ROC-AUC, and confusion matrix. Plots include feature importance, ROC curve, and model comparison. These are saved under artifacts/ and displayed in the Model Insights page.

Future Improvements

  • Hyperparameter tuning (e.g. GridSearchCV or Optuna).
  • SHAP integration for per-prediction explainability in the app.
  • Retraining pipeline on a schedule or on new data.
  • Docker image and one-click deploy (e.g. Render, Railway).
